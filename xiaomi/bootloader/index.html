2. Boot Flow Overview

Preloader (BootROM) → LK (lk_a) → Boot image (boot_a) → Android

LK Startup Sequence

  1. Exception vectors at 0x200 (B to handlers)
  2. Early init (ARM32):
      - Read CP15 system control register
      - Disable L1 cache, MMU
      - Self-relocation: copy image from load address to link address 0x48200000
      - Jump to relocated entry 0x48200080
  3. Stack init: Configure SP for IRQ, FIQ, SVC, ABT, UND modes
  4. Platform init: MT6877 clocks, PMIC, UART, display
  5. Image verification: Verify boot/recovery images via AVB 2.0
  6. Boot state management: Read SECCFG, set verifiedbootstate cmdline
  7. Kernel boot: Load and jump to kernel

4. MiCustom Layer

Xiaomi implements a custom unlock layer (separate from standard AOSP fastboot unlock). The MiCustom functions are linked as a separate module in LK.

API Surface

│ miunlock verity()         → verifies token      │
│ miunlock set sigdata()    → stores signature    │
│ mi_check_magic()          → checks RPMB magic   │
│ mi_get_magic()            → reads RPMB magic    │
│ mi_set_magic()            → writes RPMB magic   │

│ mi_set_sig()              → writes signature    │
│ Custom_error_process()    → error handler       │
└─────────────────────────────────────────────────┘

Function Details

mi_get_lock_state()

  • File offset: strings at 0xa8130
  • Calls MiCustom Get LockState → reads from SECCFG
  • Returns current lock state (0=locked, 1=unlockable, 2=unlocked)

mi_set_lock_state(state)

  • File offset: strings at 0xa811c
  • Calls MiCustom Set LockState → writes to SECCFG
  • Validates state transition: "%s want to set wrong lock state : %d" (0xa8488)
  • On failure: Custom_error_process reporting seccfg_state

mi_gen_unlock_token()

  • File offset: strings at 0xa86c8
  • Generates unlocking token bound to device identity
  • Uses:
      - hash(ufsid) — UFS device ID hash (0xa82b0)
      - cpuid — CPU unique ID (0xa8300, 0xa8340)
      - RSA decryption (rsa_exptmod at 0xa8200)
      - Base64 encoding for token format (0xa8540)
  • Token contains signature over device identifiers

miunlock verity()

  • File offset: 0xa8524
  • Verifies unlock token signature
  • Checks token SN verify success (0xa8388) → serial number verification
  • Validates sig_len and signature data
  • Returns miunlock verity failed, result=0x%x on error
  • Returns wrong unlock data (0xa84c4) for invalid format
  • Returns invalid argument for miunlock (0xa84d8) for bad args

MiCustom Token Flow

fastboot oem unlock <token> │ ├─ miunlock verity() │ ├─ Decode base64 token │ ├─ Extract signature and device SN │ ├─ RSA verify: hash(device_id) == decrypted(signature) │ │ ├─ RSA modulus: stored in binary at 0xa85c0 (272 bytes) │ │ └─ RSA exponent: 0x10001 (65537, standard) │ ├─ "token SN verify success" or "token sig not same" │ └─ Return result │ ├─ miunlock set sigdata() │ └─ Store verified signature in RPMB for audit │ └─ mi_set_lock_state(UNLOCKED)

6. ATF TrustZone Interaction

The bootloader interacts with ARM Trusted Firmware (ATF) via SMC (Secure Monitor Call) instructions.

SMC Protocol

  • Call type: Fast SMC (not Yielding)
  • SMC immediate used: 0xF0 (most common), others: 0xD0, 0xC0, 0xA0, 0xB0
  • Function IDs stored in literal pools:
      - 0xC2000000 — MTK custom fast SMC
      - 0x82000000 — Standard fast SMC (OEM/service)
  • Transition strings:
      - "[Sec] We are going to enter ATF for smc call" (0xa7e1f)
      - "[Sec] We are back from ATF" (0x1383f5)

set_hw_locked_in_atf() Function

  • String at: 0x12b290
  • Purpose: Locks/unlocks the hardware fuse via ATF
  • Makes SMC call to ATF to set hardware lock state
  • Called during both lock and unlock operations
  • SMC call parameters:
      - R0 = SMC function ID (0xC20000xx or 0x820000xx)
      - R1 = lock state to set (0 = unlock, 1 = lock)
      - R2+ = additional parameters

ATF SMC Locations

Key SMC calls are found throughout the code:

  • Early boot: 0x15a8, 0x1b7c, 0x1c00+ (series of SMCs at boot)
  • Platform init: 0x33b8, 0x3a24, 0x3d96 (DEVAPC, power management)
  • SPM (System Power Management): 0x6c34+ (SPM SMC calls)
  • SECCFG operations: 0x1bed88+ (SECCFG write via SMC)
  • Unlock operation: 0x12403c area (near SECCFG unlock functions)

10. Complete Unlock Flow Sequence

Phase 1: Command Reception
fastboot oem unlock [data|token] ├─ fastboot parses command ├─ Matches "-fastboot_oem_lock" command handler └─ Enters unlock handler with [data] parameter
Phase 2: Permission Check
├─ Check unlock_ability from SECCFG │ ├─ Read unlock_allowed_flag_offset │ └─ If unlock_ability == 0 → "Failed to get unlock permission" → DENY ├─ Read device lock state │ ├─ mi_get_lock_state() │ │ └─ MiCustom Get LockState → seccfg_get_lock_state() │ └─ If already unlocked → no action needed └─ If locked with unlock ability → continue
Phase 3: User Confirmation (if no token)
├─ Display: "Unlock bootloader?" ├─ Warning: data will be wiped ├─ Wait for volume key input │ ├─ Volume UP → continue unlock │ └─ Volume Down / timeout → cancel └─ Power-off countdown on timeout
Phase 4: Token Verification
├─ miunlock verity(unlock_data) │ ├─ Validate input: "wrong unlock data" │ ├─ Decode base64 token │ ├─ Extract serial number + signature │ ├─ Compute hash(ufsid + cpuid + serial) │ ├─ RSA verify: signature == hash │ │ ├─ rsa_exptmod(public_key, signature) │ │ ├─ padding_unpad(result) │ │ └─ compare(hash, result) │ ├─ "token SN verify success" → PASS │ └─ "token sig not same" → FAIL → "Unlock failed - Err:0x%x" └─ On verify failure → abort
Phase 5: Data Wipe
├─ unlock_erase() │ ├─ Wipe userdata partition │ └─ "factory data reset" ├─ Display: "To prevent unauthorized access to your personal data,unlocking the bootloader" └─ Clear personal data
Phase 6: State Transition
├─ mi_set_lock_state(STATE_UNLOCKED) │ ├─ Validate state transition │ ├─ seccfg_set_lock_state(UNLOCKED) │ │ ├─ Read SECCFG block │ │ ├─ Validate magic + checksum │ │ ├─ Write unlock state │ │ ├─ Write backup copy │ │ └─ Verify write ("Major SECCFG_VALID", "Bak SECCFG_VALID") │ └─ Error: "Seccfg set lock state failed!" / "Custom set lock state failed!" ├─ miunlock set sigdata(token_signature) │ └─ Store unlock token in RPMB audit trail └─ "Set_RTC_Fastboot_Mode" → RTC flag for next boot
Phase 7: ATF Hardware State
├─ set_hw_locked_in_atf(0) // 0 = unlocked │ ├─ Prepare SMC call to TrustZone │ ├─ "[Sec] We are going to enter ATF for smc call" │ ├─ SMC #x → ATF secure monitor │ │ ├─ FID: 0xC20000xx / 0x820000xx │ │ ├─ Args: unlock_state, flags │ │ └─ ATF writes hardware fuse/register │ └─ "[Sec] We are back from ATF" └─ Hardware lock state now reflects unlocked
Phase 8: Boot State Update
├─ Set kernel cmdline: │ └─ androidboot.verifiedbootstate=orange ├─ AVB state: allow_unverified_slots = true ├─ Set lock_state in SECPOLICY │ └─ "[SEC_POLICY] lock_state = UNLOCKED" └─ Display: orange_state_warning at boot logo
Phase 9: Reboot
├─ Save RTC fastboot flag ├─ Reboot device └─ LK reloads → detects unlocked state ├─ Skips AVB verification ├─ Sets ORANGE boot state └─ Allows custom images
