1. Архитектура бинарного файла

Бинарный файл LK — это raw binary со специфичным для Mediatek заголовком:

0x0304Magic/ID2: 0x58891689

Последовательность запуска LK

  1. Векторы исключений по адресу 0x200 (B к обработчикам)
  2. Ранняя инициализация (ARM32):
      - Чтение регистра управления системы CP15
      - Отключение L1 кэша, MMU
      - Самоперемещение: копирование образа из адреса загрузки в адрес привязки 0x48200000
      - Переход на перемещённую точку входа 0x48200080
  3. Инициализация стека: Настройка SP для режимов IRQ, FIQ, SVC, ABT, UND
  4. Инициализация платформы: тактирование MT6877, PMIC, UART, дисплей
  5. Верификация образов: Проверка образов boot/recovery через AVB 2.0
  6. Управление состоянием загрузки: Чтение SECCFG, установка cmdline verifiedbootstate
  7. Загрузка ядра: Загрузка и переход к ядру

4. Слой MiCustom

Xiaomi реализует собственный слой разблокировки (отдельно от стандартного AOSP fastboot unlock). Функции MiCustom скомпонованы как отдельный модуль в LK.

API поверхность

◆ MiCustom Interface

Детали функций

mi_get_lock_state()

  • Смещение в файле: строки по адресу 0xa8130
  • Вызов MiCustom Get LockState → чтение из SECCFG
  • Возвращает текущее состояние блокировки (0=заблокирован, 1=разблокируемый, 2=разблокирован)

mi_set_lock_state(state)

  • Смещение в файле: строки по адресу 0xa811c
  • Вызов MiCustom Set LockState → запись в SECCFG
  • Проверяет переход состояния: "%s want to set wrong lock state : %d" (0xa8488)
  • При ошибке: Custom_error_process с сообщением seccfg_state

mi_gen_unlock_token()

  • Смещение в файле: строки по адресу 0xa86c8
  • Генерирует токен разблокировки, привязанный к идентификатору устройства
  • Использует:
      - hash(ufsid) — хеш ID UFS устройства (0xa82b0)
      - cpuid — уникальный ID CPU (0xa8300, 0xa8340)
      - RSA дешифрование (rsa_exptmod по адресу 0xa8200)
      - Base64 кодирование для формата токена (0xa8540)
  • Токен содержит подпись идентификаторов устройства

miunlock verity()

  • Смещение в файле: 0xa8524
  • Проверяет подпись токена разблокировки
  • Проверяет token SN verify success (0xa8388) → верификацию серийного номера
  • Проверяет sig_len и данные подписи
  • Возвращает miunlock verity failed, result=0x%x при ошибке
  • Возвращает wrong unlock data (0xa84c4) при неверном формате
  • Возвращает invalid argument for miunlock (0xa84d8) при плохих аргументах

Поток токена MiCustom

fastboot oem unlock <token>
miunlock verity()
Декодирование base64 токена
Извлечение подписи и SN устройства
RSA проверка: hash(device_id) == decrypted(signature)
RSA modulus: сохранён в бинарном файле по адресу 0xa85c0 (272 байта)
RSA exponent: 0x10001 (65537, standard)
"token SN verify success" or "token sig not same"
Возврат результата
miunlock set sigdata()

6. Взаимодействие с ATF TrustZone

Загрузчик взаимодействует с ARM Trusted Firmware (ATF) через инструкции SMC (Secure Monitor Call).

Протокол SMC

  • Тип вызова: Fast SMC (не Yielding)
  • Используемый SMC immediate: 0xF0 (наиболее частый), другие: 0xD0, 0xC0, 0xA0, 0xB0
  • ID функций, хранящиеся в пулах литералов:
      - 0xC2000000 — MTK пользовательский fast SMC
      - 0x82000000 — Стандартный fast SMC (OEM/сервис)
  • Строки перехода:
      - "[Sec] We are going to enter ATF for smc call" (0xa7e1f)
      - "[Sec] We are back from ATF" (0x1383f5)

Функция set_hw_locked_in_atf()

  • Строка по адресу: 0x12b290
  • Назначение: Блокировка/разблокировка аппаратного fuse через ATF
  • Выполняет SMC вызов к ATF для установки состояния аппаратной блокировки
  • Вызывается как при блокировке, так и при разблокировке
  • Параметры SMC вызова:
      - R0 = SMC function ID (0xC20000xx or 0x820000xx)
      - R1 = lock state to set (0 = unlock, 1 = lock)
      - R2+ = additional parameters

Расположения SMC ATF

Ключевые SMC вызовы найдены по всему коду:

  • Early boot: 0x15a8, 0x1b7c, 0x1c00+ (series of SMCs at boot)
  • Platform init: 0x33b8, 0x3a24, 0x3d96 (DEVAPC, power management)
  • SPM (System Power Management): 0x6c34+ (SPM SMC calls)
  • SECCFG operations: 0x1bed88+ (SECCFG write via SMC)
  • Unlock operation: 0x12403c area (near SECCFG unlock functions)

ATF SMC Function IDs:

FIDLocationPurpose
0xC20000000x1c1dacMTK custom SMC base
0x820000000x1c44c0+Fast SMC (OEM/service)
Sub-FIDsVariousLock/unlock, SEJ, RPMB

Аппаратные fuse блокировки

Безопасный мир ATF управляет аппаратными fuse (efuse) для:

  • SBC (Secure Boot Control): Аппаратное обеспечение безопасной загрузки
  • SDL (Security Debug Lock): Контроль доступа к отладочному порту
  • Читаются через: dump sw sbc:0x%x, sw sdl:0x%x , hw sbc: %d (0x136fb8)

7. RPMB

RPMB (Replay Protected Memory Block) используется для:

  1. Программирование ключа аутентификации — хранение общего секрета
  2. Конфигурация защиты записи — предотвращает вмешательство в разделы
  3. Аудит разблокировки — хранение доказательств подписи разблокировки

9. Криптография

Открытый ключ RSA

Верификация токена разблокировки MiCustom использует RSA ключи длиной 2048 бит.

Потенциальный RSA modulus (2048-бит / 256 байт) хранится по адресу 0xa85c0:

Этот ключ используется при верификации токена: "Error: digital number (%d) of rsa n is smaller than that (%d) of input data" и "Error: fail to read rsa n data".

Additional components:

  • rsa_exptmod — функция модульного возведения в степень RSA (0xa8200)
  • RSA_padding_add — проверка PKCS padding
  • tmp_key параметр для операций RSA

Открытый ключ OEM

g_sec_oemkey (в файле по адресу 0x137bdc) — глобальная переменная, хранящая открытый ключ OEM, используемый для проверки подписанных образов.

HMAC

HMAC используется для аутентификации RPMB:

  • "rpmb_hmac error" — ошибка HMAC верификации
  • "Authentication failure(MAC not match)" — ошибка MAC проверки
  • "compare hmac error" — несовпадение при сравнении

Структура токена

Unlock token format
Base64 encoded
Contains: device SN + RSA signature
Size: sig_len variable ("no sig stored", "sig_len = %d")
Verified against: hash(ufsid) + cpuid + serial number

Хеш

SHA-256 используется для различных операций:

  • "ERROR: CAN NOT SHA256(UFSID)" — хеширование ID UFS устройства
